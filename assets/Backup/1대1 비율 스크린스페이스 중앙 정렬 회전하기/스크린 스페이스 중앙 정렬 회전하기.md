개요


특정 해상도 / 비율에서는 스크린 이펙트의 중앙 정렬이 동작하지 않는 문제가 있습니다.

세로가 가로보다 특정 값 이상보다 긴 경우 (아이폰의 9 : 20 비율 등) 확실하게 눈에 띄게됩니다.

원인 파악


화면 가로 세로 비율에 맞춰서 스크린 스페이스의 중앙을 고정하는 것은 이전 작업에서 성공했으나, 이를 같은 비율을 유지하며 회전시키는 것은 다른 차원의 문제였습니다.

 



가로 세로 비율에 맞춰서 편집된 스크린 스페이스를 회전시키면, 회전시키는 중점인 앵커의 위치가 실제 스크린 스페이스의 바깥에 위치하게 되기 때문에, 이를 비율에 맞춰서 보정해야 합니다.

 



왜냐하면 가로 세로 비율으로 보정한 스크린 스페이스는 항상 정사각형의 비율을 유지하도록 만들어졌기 때문입니다. 비율을 계산해서 세로가 길면 그만큼 가로도 길어지고, 반대도 마찬가지입니다.

실제로 UV를 Fract으로 끊어보면 아무리 비율을 조절해도 1을 넘는 값이 없습니다.

해결 방안
정사각형의 비율을 유지한 상태로 UV 오프셋을 이용해 가로 세로의 비율만큼 UV를 화면 중앙으로 오프셋 해서 해결합니다.

 



비율에 따른 오프셋 값을 계산하기 위해서 각각 세로/가로 비가 1/5, 1/4인 경우의 수를 준비했습니다.

여기서 빨간색 영역으로 표시된 부분은 실제 화면이며, 파란색 영역은 실제 UV입니다.

1/5의 경우 1/5 * 0.5만큼 곱한 값이 앵커가 되어야 합니다.
1/4의 경우 1/4 * 0.5만큼 곱한 값이 앵커가 되어야 합니다.
이것은 가로가 더 긴 경우에만 해당하며, 세로 축의 UV가 얼마나 움직여야 하는지를 나타내지만,

실제로는 위의 빨간 점선 화살표 만큼 움직여야 파란색 영역만큼 UV 전체가 오프셋 된다는 문제가 있습니다.

 



그래서 먼저 UV를 -0.5 만큼 오프셋 합니다. 

이렇게 하면 항상 앵커는 화면의 바닥에 오게됩니다.

 



그 다음 가로 세로 비율에 맞춰서 1/4 * 0.5만큼 다시 위로 올립니다.

이렇게 계산하면 어떤 비율을 가진 화면이든, 빨간색 영역으로 표시된 부분의 실제 화면의 정중앙에 UV가 위치하게 되는 것입니다.

작업 내용

float2 ScreenRatio()
{
    // x = width
    // y = height
    // z = 1 + 1.0/width
    // w = 1 + 1.0/height
    float2 ratio = 1;
    
    if(_ScreenParams.x > _ScreenParams.y)
    {
        ratio = float2(1, _ScreenParams.y / _ScreenParams.x);
    }
    else
    {
        ratio = float2(_ScreenParams.x / _ScreenParams.y, 1);
    }
    
    return ratio;
}

float2 ScreenOffset()
{
    // x = width
    // y = height
    // z = 1 + 1.0/width
    // w = 1 + 1.0/height
    float2 ratio = 1;
    
    if(_ScreenParams.x > _ScreenParams.y)
    {
        ratio = float2(0, 0.5 * (_ScreenParams.y / _ScreenParams.x));
    }
    else
    {
        ratio = float2(0.5 * (_ScreenParams.x / _ScreenParams.y), 0);
    }

    return ratio;
}
이를 스크린 오프셋 함수로 제작했습니다. 오프셋이기 때문에 한 축에만 영향을 주는 식으로 제작되었습니다.

 

if (x > y)
{
    R = y/x;
}
else
{
    R = x/y;
}
가로와 세로중 항상 더 긴 쪽을 분모로 계산 합니다.

 



if(U > V)
{
    V - 0.5 + (R * 0.5);
}
else
{
    U - 0.5 + (R * 0.5);
}
UV는 반대로 더 짧은 쪽에서 -0.5로 오프셋하여 화면의 바닥에 오게 만듭니다.

그 다음 비율에 0.5를 곱한 값만큼 다시 오프셋해서 계산합니다.

결과물

어떤 비율에서도 중앙 정렬이 동작하는 것을 확인 하실 수 있습니다.

Shader "MMN/Special/CameraFade & Vignetting"
{
	Properties
	{
		[HideInInspector] _Mode("BlendMode", Float) = -1
		[Enum(Fade, 0, Vignetting, 1)]_ScreenFXMode("Screen FX Mode", Float) = 0
		_VignettingSmooth("Vignetting Smooth", Range(0, 1)) = 1
		_VignettingRange("Vignetting Range", Range(0, 1)) = 0.5

		_Color("BaseColor", Color) = (1, 1, 1, 1)
		_Intensity_Color("Intensity_Color", Float) = 1
		_Intensity_Alpha("Intensity_Alpha", Float) = 1

		[Header(Rendering Options)]
		[Space()]
		[Enum(UnityEngine.Rendering.BlendMode)]_BlendSrc("Blend Src", Float) = 5
		[Enum(UnityEngine.Rendering.BlendMode)]_BlendDst("Blend Dst", Float) = 10
	}

	SubShader
	{
		LOD 100
		Tags
		{
			"RenderPipeline" = "UniversalPipeline"
			"RenderType" = "Transparent"
			"Queue" = "Transparent+1050"
		}

		HLSLINCLUDE
			#pragma target 4.5
		ENDHLSL

		Pass
		{
			Name "Unlit"

            Tags { "LightMode" = "ScreenSpaceRenderObjects" }

			Cull back
			Blend [_BlendSrc] [_BlendDst]
			ZTest Always
			ZWrite Off
			ColorMask RGBA

			HLSLPROGRAM
			#pragma exclude_renderers glcore gles gles3

			#pragma vertex vert
			#pragma fragment frag

			#include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Core.hlsl"

			CBUFFER_START(UnityPerMaterial)
				half _Mode;
				half _ScreenFXMode;

				half _Intensity_Color;
				half _Intensity_Alpha;
				half4 _Color;

				half _VignettingSmooth;
				half _VignettingRange;
			CBUFFER_END

			struct Attributes
			{
				float4 positionOS 	: POSITION;
				float4 color 		: COLOR;
				float4 uv			: TEXCOORD0;
			};

			struct Varyings
			{
				float4 positionCS 	: SV_POSITION;
				half4 color 		: COLOR;
				float4 uv 			: TEXCOORD0;
				float4 screenPos 	: TEXCOORD1;
				float3 positionWS	: TEXCOORD2;
			};


			Varyings vert(Attributes input)
			{
				Varyings output = (Varyings)0;

				VertexPositionInputs vertexInput = GetVertexPositionInputs(input.positionOS.xyz);
				output.positionCS = vertexInput.positionCS;
				output.positionWS = TransformObjectToWorld(input.positionOS.xyz);
				output.color = input.color;
				output.uv = input.uv;
				output.screenPos = ComputeScreenPos(output.positionCS);

				return output;
			}

			void ScreenRatio(inout float2 ratio)
			{
				// x = width
				// y = height
				// z = 1 + 1.0/width
				// w = 1 + 1.0/height

				if(_ScreenParams.x > _ScreenParams.y)
				{
					ratio = float2(1, _ScreenParams.y / _ScreenParams.x);
				}
				else
				{
					ratio = float2(_ScreenParams.x / _ScreenParams.y, 1);
				}
			}

			half4 frag(Varyings input) : SV_Target
			{
				// 프로퍼티 범위 조절
				_VignettingSmooth = 1 - _VignettingSmooth;
				_VignettingSmooth *= 0.5;
				_VignettingRange *= 2;
				_VignettingRange -= 1;

				half3 color = 1;
				half alpha = 1;

				half3 fadeColor = _Color.rgb * _Intensity_Color * input.color.rgb;
				half fadeAlpha = saturate(_Intensity_Alpha * _Color.a * input.color.a);

				float2 ratio = 1;
				ScreenRatio(ratio);

				float2 screenUV = float2(input.screenPos.x, input.screenPos.y) * ratio;
				screenUV = (screenUV / input.screenPos.w + 0.0001) - 0.5 * ratio;

				// // Quad Mask
				// half2 quadMaskUV = 1 - saturate(abs(screenUV) * 2);
				// half quadMask = saturate(quadMaskUV.x * quadMaskUV.y + _VignettingRange);
				// quadMask = 1 - saturate(smoothstep(_VignettingSmooth, 1 - _VignettingSmooth, quadMask));

				// Length Mask
				half lengthMask = length(screenUV) - _VignettingRange;
				lengthMask = smoothstep(_VignettingSmooth, 1 - _VignettingSmooth, lengthMask);

				// // Default Mask
				// half defualtMask = saturate(abs(screenUV.y) + abs(screenUV.x) + _VignettingRange);
				// defualtMask = smoothstep(_VignettingSmooth, 1 - _VignettingSmooth, defualtMask);

				half3 vignettingColor = lengthMask;
				half vignettingAlpha = lengthMask;

				color = lerp(fadeColor, vignettingColor * fadeColor, _ScreenFXMode);
				alpha = lerp(fadeAlpha, vignettingAlpha * fadeAlpha, _ScreenFXMode);

				half4 finalColor = half4(color, alpha);

				return finalColor;
			}
			ENDHLSL
		}
	}
	Fallback off
	CustomEditor "MM.Client.Editor.ShaderGUI.MMN_ScreenFxGUI"
}
